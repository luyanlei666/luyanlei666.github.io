QueryWrapper中常用方法总结

QueryWrapper中常用方法总结

函数名 说明 列子
eq 等于= 例：eq(“name”,”张三”) —> SQL中：name = ‘张三’
ne 不等于<> 例：ne(“name”,”张三”) —> SQL中：name <> ‘张三’
gt 大于> 例：gt(“age”,18) —> SQL中：age > 18
ge 大于等于>= 例：ge(“age”,18) —> SQL中：age >= 18
lt 小于< 例：lt(“age”,20) —> SQL中：age < 20
le 小于等于<= 例：le(“age”,20) —> SQL中：age <= 20
between BETWEEN 值1 AND 值2 例：between(“age”,20,30) —> SQL中：age between 20 and 30
notBetween not BETWEEN 值1 AND 值2 例：notBetween(“age”,20,30) —> SQL中：age not between 20 and 30
like LIKE ‘%值%’ 例：like(“name”,”李”) —> SQL中：name like ‘%李%’
notLike not like ‘%值%’ 例：notLike(“name”,”李”) —> SQL中：name not like ‘%李%’
likeLeft like ‘%值’ 例：likeLeft(“name”,”%磊”) —> SQL中：name like ‘%磊’
likeRighe like ‘值%’ 例：likeLeft(“name”,”王%”) —> SQL中：name like ‘王%’
isNull 字段为null 例：isNull(“name”) —> SQL中：name is null
isNotNull 字段不为空 例：isNotNull(“name”) —> SQL中：name is not null
in 字段IN(v0，v1，…) 例：in(“age”,{10,15,18}) —> SQL中：age in (10,15,18)
notIn 字段NOTIN(v0，v1，…) 例：notIn(“age”,{10,15,18}) —> age not in(10,15,18)
inSql 字段 IN（sql语句） 例：insql(“id”,”select id from table where id < 3”) —> SQL中：id in(select id from table where id < 3)
notInSql 字段 NOT IN （Sql语句） 例：notInSql(“id”,”select id from table where id < 3”) —> SQL中：id not in(select id from table where id < 3)
groupBy 分组：GROUP BY 字段… 例：groupBy(“id”,”name”) —>SQL中：group by id,name
orderByAsc 排序：ORDER BY 字段…ASC 例：orderByAsc(“id”) —> SQL中：order by id ASC
orderByDesc 排序：ORDER BY 字段… DESC 例：orderByDesc(“id”,”name”) —> order by id DESC,name DESC
orderBy 排序：ORDER BY 字段… 例：orderBy(true,true,”id”,”name”) —> order by id ASC,name ASC
having HAVING(sql语句) 例：having(“sum(age) > {0}”,11) —>SQL中： having sum(age) >11
or 拼接OR 注意事项：主动调用or表示紧接着下一个方法不是用and连接!(不调用or则默认为使用and连接)；例：eq(“id”,1).or().eq(“name”,”张三”) —> SQL中：id=1 and name = ‘张三’
and AND嵌套 例：and(i->i.eq(“name”,”张三”).ne(“status”,”活着”)) —> SQL中：and（name=‘张三’ and status <> ‘活着’）
apply 拼接sql 注意事项：该方法可用于数据库函数，动态入参的params对应前面sqlHaving内部的{index}部分，这样是不会sql注入风险的，反之会有！例：apply(“date_format(dateCokumn,’%Y-%m-%d’)={0}”,”2021-09-17”) —> SQL中：date_format(dateColumn,’%Y-%m-%d’) = ‘2021-09-17’
last 务实优化规则直接凭借到SQL的最后 注意：只能调用一次，多次调用以最后一次为准，有sql注入的风险，请谨慎使用。例：last(“limit 1”)
exists 拼接EXISTS(sql语句) 例：exists(“select id from table where age =33”) —> exists(“select id from table where age = 1”)
notExists 拼接NOT EXISTS(sql语句) 例：notExists(“select id from table where age =33”) —> not exists(“select id from table where age = 1”)
nested 正常嵌套不带AND或者OR 例：nested(i->i.eq(“name”,”张三”).ne(“status”,”正常”)) —> SQL中：(name = ‘张三’ and status <> ‘正常’)

WebRtc遇到的问题总结

WebRtc遇到的问题总结

第一次接触webrtc，遇到了各式各样的问题，期间上网查询了相关资料，但网上关于这方面的很少，通过自己搭建环境测试，将这个过程中遇到的问题以及解决的办法记录下来。

1. 浏览器抛出：doAnswerBOdy setRemoteDescription failed : OperationError:Failed to parse SessionDescription . a=extmap-allow-mixed Expects at least 2 fields 问题。

原因：谷歌浏览器版本问题，谷歌浏览器自M71版本开始支持SDP协议属性 a=extmap-allow-mixed，M71之前版本Chrome的SDP协商将会失败。从M89版本开始，a=extmap-allow-mixed将被默认提供。所以，一旦出现这种提示，说明通话两端的浏览器有一方使用的版本低于M71.
解决办法：将低版本浏览器升级，或将高版本浏览器降级（低于M71），就能实现通话。

2. 出现 bad ice candidate(OperationError) 具体错误信息如下图：


原因：当不同网段进行视频通话时(即两端处于不同网络)，需要配置中转服务器stun/turn,出现bad ice candidate 可以将问题定位在中转服务器stun/turn上。
具体办法：查看stun/turn服务是否开启，如果没有启动，启动服务即可：turnserver -o -a -f -user=zxs:123456 -r guangdong ；如果已启动，利用cmd，去ping一下stun/turn服务器ip，查看是否能ping通，以及丢包率情况。
